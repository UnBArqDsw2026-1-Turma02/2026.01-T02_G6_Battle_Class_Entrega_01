# Check-list de Verificação - Design Sprint (Battle Class)

## 1. Entendimento do Problema

- [x] O problema está claramente definido? → Sim  
   Justificativa: O grupo definiu como problema central a baixa motivação e engajamento dos estudantes em métodos tradicionais de estudo.

- [x] O público-alvo foi identificado? → Sim  
   Justificativa: O foco está em estudantes que se preparam para provas, como ENEM, Concursos e demais vestibulares.

- [x] Foram levantadas dores reais dos usuários? → Parcial  
   Justificativa: As dores foram discutidas com base em experiências do grupo, mas sem pesquisa formal com usuários externos.

- [x] Existe um objetivo claro para a solução? → Sim  
   Justificativa: O objetivo é tornar o estudo mais engajador por meio de gamificação.

---

## 2. Ideação

- [x] Foram geradas múltiplas ideias de solução? → Sim  
   Justificativa: O grupo explorou diferentes abordagens antes de decidir uma ideia norteadora.

- [x] As ideias consideram diferentes abordagens? → Sim  
   Justificativa: Foram avaliadas alternativas como competição direta e modos individuais com progressão.

- [x] Houve participação ativa do grupo na ideação? → Sim  
   Justificativa: Todos os integrantes contribuíram com sugestões durante a construção da solução.

- [x] As ideias foram documentadas (ex: sketches, notas)? → Parcial  
   Justificativa: As ideias foram registradas principalmente em texto, com pouca utilização de sketches visuais.

---

## 3. Decisão

- [x] Existe um critério claro para escolha da solução? → Parcial  
   Justificativa: A decisão considerou viabilidade e simplicidade, mas sem uso de métodos formais estruturados.

- [x] A solução escolhida resolve o problema principal? → Sim  
   Justificativa: A proposta utiliza gamificação para aumentar o engajamento no processo de estudo.

- [x] O grupo chegou a um consenso? → Sim  
   Justificativa: A decisão final foi acordada entre todos os integrantes após discussão.

- [x] Foi definida uma proposta clara de solução? → Sim  
   Justificativa: A solução foi definida como um sistema com duas fases: perguntas e tower defense.

---

## 4. Prototipação

- [x] O protótipo representa o fluxo principal do sistema? → Sim  
   Justificativa: O fluxo contempla responder perguntas, ganhar moedas e utilizá-las no jogo.

- [x] O nível de fidelidade é adequado? → Sim  
   Justificativa: Foi desenvolvido um protótipo de baixa fidelidade e alta fidelidade, adequado para validação inicial.

- [x] As principais funcionalidades estão representadas? → Sim  
   Justificativa: Estão presentes as funcionalidades essenciais como perguntas, moedas e jogo.

- [x] O protótipo é navegável/simulável? → Sim  
   Justificativa: O protótipo permite visualizar o fluxo e interagir também.

---

## 5. Testes com Usuários

- [x] O protótipo foi testado com usuários reais? → Sim  
   Justificativa: Foram realizados testes formais com usuários que representam o público alvo desse projeto.

- [x] Foram coletados feedbacks relevantes? → Sim  
   Justificativa: Os testes permitiram identificar melhorias no fluxo e na clareza da proposta.

- [x] Os usuários entenderam a proposta do sistema? → Sim  
   Justificativa: De forma geral, os usuários compreenderam a dinâmica de perguntas e jogo.

- [x] Foram identificados pontos de melhoria? → Sim  
   Justificativa: Foram levantadas melhorias relacionadas à usabilidade e clareza das mecânicas.

---

## 6. Coerência da Solução

- [x] A solução resolve o problema inicial? → Sim  
   Justificativa: A gamificação proposta atua diretamente no aumento do engajamento dos usuários.

- [x] O fluxo do usuário é lógico e intuitivo? → Sim  
   Justificativa: O fluxo segue uma sequência clara: estudar, ganhar recursos e jogar.

- [x] Existe consistência entre ideia, protótipo e objetivo? → Sim  
   Justificativa: Todos os elementos estão alinhados com a proposta de aprendizado gamificado.

- [x] A solução é viável para implementação futura? → Sim  
   Justificativa: O escopo foi reduzido para um MVP viável tecnicamente.

---

## 7. Documentação

- [x] Todas as etapas do Design Sprint foram registradas? → Sim  
   Justificativa: As etapas foram documentadas ao longo do desenvolvimento do projeto.

- [x] As decisões do grupo estão documentadas? → Sim  
   Justificativa: As escolhas realizadas estão descritas no material produzido.

- [x] O protótipo está acessível? → Sim  
   Justificativa: O protótipo foi disponibilizado por meio de link para visualização.

- [x] O material está organizado e compreensível? → Sim  
   Justificativa: O conteúdo está estruturado de forma clara e lógica para leitura.
