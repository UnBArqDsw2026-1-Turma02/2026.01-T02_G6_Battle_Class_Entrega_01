## Etapa 1 - Unpack (Entender)

### Definição da Stack Tecnológica

O grupo iniciou debatendo as tecnologias a serem utilizadas. Cada membro trouxe sua perspectiva:

- João Lobo sugeriu Java para o back-end;
- Otávio e Lucas propuseram Flutter;
- Dannyeclisson se posicionou contra React Native.

Após debate, o grupo convergiu para uma solução **web mobile-first**, equilibrando familiaridade técnica e escopo da disciplina:

| Camada | Tecnologia Escolhida |
|--------|---------------------|
| Frontend | React (Mobile First) |
| Backend | Express + TypeScript |
| Banco de Dados | PostgreSQL via Supabase |

### Brainstorming do Conceito do Jogo

O grupo realizou uma sessão de brainstorming para definir a mecânica central do produto. Foram exploradas três iterações:

**Ideia 1 — Batalha em tempo real:**
Jogo multiplayer onde cada pergunta acertada dispara um ataque ao adversário em tempo real. Descartada pelo grupo pois o tempo de resposta das perguntas tornaria a experiência inviável e a complexidade de sincronização estaria fora do escopo.

**Ideia 2 — Batalha assíncrona com fase de perguntas prévia:**
As perguntas seriam respondidas antes de cada batalha, acumulando recursos para o combate contra outro usuário. Descartada por ser pesada demais para o escopo da disciplina.

**Ideia Final (aprovada por todos):**
As perguntas são respondidas antes de cada jogo para acumular moedas. Com as moedas, o jogador enfrenta ondas de inimigos em um jogo de **tower defense**. O formato de jogo competitivo ficou em aberto para iterações futuras.

### É/Não É · Faz/Não Faz

Para alinhar expectativas de escopo, o grupo realizou a dinâmica **É/Não É · Faz/Não Faz**, consolidando o que o produto é e o que está fora do escopo desta entrega.

## Etapa 3 - Decision (Decidir)

O grupo realizou uma reunião para avaliar os Rich Pictures produzidos na etapa anterior e escolher a visão que melhor representava o sistema.

### Rich Picture Escolhido

O grupo escolheu o Rich Picture da **Marina** como base, por ser o mais completo, lúdico, didático e por consolidar os pontos discutidos no Unpack. A partir dele, foram feitas discussões sobre os fluxos do sistema, gerando uma segunda versão refinada.

| v2 — Refinado | Versão final após discussões de fluxo | ![v2](../assets/design_sprint/rich_pictures/Rich_picture_final.jpeg) |

### Ata da Reunião

![Reunião Decision](../assets/design_sprint/reuniao_decision.png)

O grupo alinhou envio de artefatos, status de protótipos e prazos para entrega. Foram tomadas as seguintes decisões:

- **Prazo de envio ao repositório**: até segunda-feira; **apresentação final**: sexta-feira.
- **BPMN** revisado com feedback positivo sobre qualidade visual e setas.
- **Distinção entre verificação e validação**: verificação como checklist interno; validação como testes com 3–5 usuários reais.
- **Responsabilidades definidas**:

| Membro | Responsabilidade |
|--------|-----------------|
| João | Protótipo de alta fidelidade |
| Marina | Rich Picture final |
| Ana Elisa | Atualização do Git/GitPages |
| Lucas | Pesquisa, persona e perguntas no Forms |
| Otávio | Compilação de decisões e documento da solução vencedora |
